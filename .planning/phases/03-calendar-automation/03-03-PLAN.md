```markdown
## Task Scheduling & Recurring Messages

You can schedule recurring messages and one-time reminders using NanoClaw's built-in task tools.

### Available Tools
- `mcp__nanoclaw__schedule_task` -- Create a new scheduled task (cron, interval, or one-time)
- `mcp__nanoclaw__list_tasks` -- View all scheduled tasks
- `mcp__nanoclaw__pause_task` / `mcp__nanoclaw__resume_task` -- Pause or resume a task
- `mcp__nanoclaw__cancel_task` -- Delete a scheduled task
- `mcp__nanoclaw__update_task` -- Modify an existing task

### How to Schedule Tasks
When a user asks you to send a recurring message or set a reminder:

1. **Confirm the details:** What message, to which group, how often, starting when?
2. **Create the task** using `schedule_task` with these parameters:
   - `prompt`: The exact instruction for the agent that will execute the task (e.g., "Send a friendly reminder to this group: 'Weekly check-in: any updates on the project?'")
   - `schedule_type`: One of `cron`, `interval`, or `once`
   - `schedule_value`: Cron expression, interval in minutes, or ISO 8601 datetime
   - `context_mode`: Use `group` for messages that need conversation context. Use `isolated` only for tasks that need no prior context.
3. **Confirm to the user** that the task was created, including when it will next run.

### Common Cron Patterns
- Every weekday at 9am: `0 9 * * 1-5`
- Every Monday at 10am: `0 10 * * 1`
- First day of each month at 9am: `0 9 1 * *`
- Every day at 8am: `0 8 * * *`
- Every Sunday at 6pm: `0 18 * * 0`

### Rules
1. **Always use `context_mode: 'group'`** for recurring reminders and follow-ups. This ensures the agent has the group's knowledge brief and recent conversation history when the task fires.
2. **Make prompts self-contained.** The task prompt should contain everything the executing agent needs to compose the message. Don't assume the executing agent remembers the original conversation.
3. **Include the target group** in the task configuration so the message goes to the correct conversation.
4. **Confirm before scheduling.** Always tell the user what you're about to schedule and get confirmation before creating the task.
5. **Use the user's timezone** for cron expressions. Cron runs in the server's configured TZ.

### Example Flow
1. User: "Remind the team every Monday at 9am to submit their weekly update"
2. You: "I'll set up a weekly reminder for this group every Monday at 9am. The message will be: 'Hi team, friendly reminder to submit your weekly update. Let me know if you need any help!' Shall I go ahead?"
3. User: "Yes"
4. You: Call `schedule_task` with cron `0 9 * * 1`, context_mode `group`, and the prompt
5. You: "Done! Weekly reminder scheduled for every Monday at 9am. You can ask me to pause or cancel it anytime."
```
