**Implementation** (`gcal-mcp-server.ts`):

Create a standalone MCP server following the exact pattern from `ipc-mcp-stdio.ts`:

```typescript
import { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';
import { z } from 'zod';
import { google } from 'googleapis';
import { GoogleAuth } from 'google-auth-library';
```

Server setup:
- `new McpServer({ name: 'gcal', version: '1.0.0' })`
- Auth: `new GoogleAuth({ keyFile: process.env.GOOGLE_CREDENTIALS_PATH, scopes: ['https://www.googleapis.com/auth/calendar'] })`
- Calendar client: `google.calendar({ version: 'v3', auth })`
- Calendar ID: `process.env.GOOGLE_CALENDAR_ID || 'primary'`

Register three tools using `server.tool()`:

1. **check_availability** -- Parameters: `date_start` (string, ISO 8601), `date_end` (string, ISO 8601), `timezone` (optional string, IANA). Calls `calendar.freebusy.query()`. Returns text content with free/busy status. Default timezone from `process.env.TZ || 'UTC'`.

2. **create_event** -- Parameters: `summary` (string), `start_time` (string, ISO 8601), `end_time` (string, ISO 8601), `timezone` (optional string), `description` (optional string), `location` (optional string), `add_meet_link` (optional boolean, default true), `attendees` (optional string array). Calls `calendar.events.insert()` with `conferenceDataVersion: 1` when Meet link requested. Returns text with event summary, time, Meet link (if created), calendar link.

3. **list_events** -- Parameters: `date_start` (optional string, defaults to now), `date_end` (optional string, defaults to 7 days from now), `max_results` (optional number, default 10). Calls `calendar.events.list()` with `singleEvents: true, orderBy: 'startTime'`. Returns formatted list or "No upcoming events found."

Connect via `new StdioServerTransport()` and `server.connect(transport)`.

Use the exact code examples from the research document (03-RESEARCH.md "Code Examples" section) as the implementation reference. Those examples are production-ready.

IMPORTANT anti-patterns to avoid:
- Do NOT use `import { google } from 'googleapis'` at top level for tree-shaking. Use it but know this is the standard pattern for the MCP server context (running in container, not bundled).
- Do NOT store credentials in code. Read from `GOOGLE_CREDENTIALS_PATH` env var only.
- Do NOT use `any` type where avoidable. Use Google Calendar API types from googleapis.
