  </read_first>
  <action>
    All commands run on EC2 via SSH.

    **Step 1: Stop NanoClaw before auth setup:**
    ```
    systemctl --user stop nanoclaw
    ```

    **Step 2: Clean any previous auth state:**
    ```
    rm -rf store/auth/
    ```

    **Step 3: Run WhatsApp pairing code authentication:**
    The user must have their phone ready at Settings > Linked Devices > Link a Device BEFORE running this command. The pairing code expires in ~60 seconds.

    ```
    rm -f store/pairing-code.txt
    npx tsx setup/index.ts --step whatsapp-auth -- --method pairing-code --phone {PHONE_NUMBER} > /tmp/wa-auth.log 2>&1 &
    ```

    Where {PHONE_NUMBER} is the TEST WhatsApp number in format 972XXXXXXXXX (country code + number, no + or spaces).

    **Step 4: Poll for pairing code:**
    ```
    for i in $(seq 1 20); do [ -f store/pairing-code.txt ] && cat store/pairing-code.txt && break; sleep 1; done
    ```

    Display the code prominently to the user. They must enter it on their phone within 60 seconds.

    **Step 5: Poll for auth completion:**
    ```
    for i in $(seq 1 60); do
      grep -q 'AUTH_STATUS: authenticated' /tmp/wa-auth.log 2>/dev/null && echo "SUCCESS" && break
      grep -q 'AUTH_STATUS: failed' /tmp/wa-auth.log 2>/dev/null && echo "FAILED" && break
      sleep 2
    done
    ```

    **Step 6: Verify auth state saved:**
    ```
    test -f store/auth/creds.json && echo "Auth state saved" || echo "Auth FAILED - retry from step 2"
    ```

    **Step 7: Register main/self-chat group:**
    ```
    npx tsx setup/index.ts --step register \
      --jid "{USER_JID}@s.whatsapp.net" \
      --name "Main" \
      --trigger "@Andy" \
      --folder "whatsapp_main" \
      --channel whatsapp \
      --assistant-name "Andy" \
      --is-main \
      --no-trigger-required
    ```
    Where {USER_JID} is the phone number (e.g., 972XXXXXXXXX).

    **Step 8: Register a test client group:**
    First, the user needs to create a WhatsApp group for testing and get its JID. The JID will appear in NanoClaw logs when a message is received in an unregistered group. Alternatively, use the list-groups setup step if available.

    ```
    npx tsx setup/index.ts --step register \
      --jid "{GROUP_JID}@g.us" \
      --name "Test Client" \
      --trigger "@Andy" \
      --folder "whatsapp_test_client" \
      --channel whatsapp
    ```

    **Step 9: Start NanoClaw:**
    ```
    systemctl --user start nanoclaw
    systemctl --user status nanoclaw
    ```

    NOTE: This task will require human interaction during steps 3-5 (phone pairing) and step 8 (providing group JID). Create a checkpoint within the task flow if the executor supports it, otherwise pause for user input at those points.
  </action>

  <what-built>WhatsApp connected to NanoClaw on EC2 with main chat and test client group registered. The assistant should respond when tagged with @Andy.</what-built>
  <how-to-verify>
    **Test 1: Basic tagged reply (main chat)**
    1. Open WhatsApp on your phone
    2. Go to your self-chat or the main conversation
    3. Send: "@Andy hello, are you working?"
    4. Wait up to 30 seconds for a reply
    5. Expected: The assistant replies with a greeting or acknowledgment

    **Test 2: Tagged reply in test client group**
    1. Open the test client WhatsApp group
    2. Send: "@Andy what can you help me with?"
    3. Wait up to 30 seconds for a reply
    4. Expected: The assistant replies (content may be generic since no knowledge brief loaded yet)

    **Test 3: No response without tag**
    1. In the test client group, send a message WITHOUT @Andy (e.g., "just a regular message")
    2. Wait 15 seconds
    3. Expected: No reply from the assistant (trigger required for non-main groups)

    **Test 4: Group isolation verification**
    1. Send "@Andy remember the code word is PINEAPPLE" in the main chat
    2. Wait for reply
    3. In the test client group, send "@Andy what is the code word?"
    4. Expected: The assistant in the test group does NOT know "PINEAPPLE" because it has separate context

    **Test 5: Session persistence**
    1. On EC2, restart NanoClaw: `systemctl --user restart nanoclaw`
    2. Wait 10 seconds
    3. Send "@Andy are you still there?" in any chat
    4. Expected: Reply comes through (WhatsApp session survived restart via stored auth state)

    **If any test fails:**
    - Check EC2 logs: `journalctl --user -u nanoclaw --since "5 min ago"`
    - Check NanoClaw logs: `cat ~/nanoclaw/logs/nanoclaw.log | tail -50`
    - Verify the assistant name matches: `grep ASSISTANT_NAME ~/nanoclaw/.env`
  </how-to-verify>
