  <name>Task 2: Bootstrap EC2 and deploy NanoClaw with systemd</name>
  <files>.env, data/env/env, ~/.config/systemd/user/nanoclaw.service</files>
  <read_first>
    - .planning/phases/01-infrastructure-whatsapp/01-RESEARCH.md (EC2 Deployment Structure, systemd Service Pattern, Anti-Patterns sections)

  <action>
    Using SSH to the EC2 instance (with IP and key from Task 1):

    1. **Clone the fork to EC2:**
       ```
       ssh -i {SSH_KEY} ec2-user@{EC2_IP} "gh auth login && gh repo clone {user}/nanoclaw ~/nanoclaw"
       ```
       If gh CLI not available on EC2, use git clone with HTTPS:
       ```
       ssh -i {SSH_KEY} ec2-user@{EC2_IP} "git clone https://github.com/{user}/nanoclaw.git ~/nanoclaw"
       ```

    2. **Run EC2 bootstrap script:**
       ```
       ssh -i {SSH_KEY} ec2-user@{EC2_IP} "cd ~/nanoclaw && bash setup/ec2-init.sh"
       ```
       This installs Docker, Node.js 22 via nvm, verifies build tools, and creates 1 GB swap.

    3. **IMPORTANT: Reconnect SSH** (required for Docker group membership):
       ```
       ssh -i {SSH_KEY} ec2-user@{EC2_IP}
       ```

    4. **Verify Docker group is active:**
       ```
       docker info > /dev/null 2>&1 && echo "Docker OK" || echo "Docker NOT READY - log out and back in"
       ```

    5. **Install dependencies and build:**
       ```
       cd ~/nanoclaw
       npm install
       npm run build
       ```

    6. **Configure environment:**
       ```
       cp .env.example .env
       ```
       Edit .env to set the real ANTHROPIC_API_KEY (from user_setup). Verify these values are present:
       - `ANTHROPIC_API_KEY=sk-ant-...` (real key from user)
       - `ASSISTANT_NAME=Andy`
       - `TZ=Asia/Jerusalem`
       - `MAX_CONCURRENT_CONTAINERS=2`
       - `ASSISTANT_HAS_OWN_NUMBER=false`

       Then sync to container env:
       ```
       mkdir -p data/env && cp .env data/env/env
       ```

    7. **Build the agent container image:**
       ```
       cd ~/nanoclaw
       bash container/build.sh || docker build -t nanoclaw-agent:latest container/
       ```
       Verify: `docker images nanoclaw-agent` shows the image.

    8. **Run NanoClaw setup (generates systemd unit + OneCLI init):**
       ```
       npx tsx setup/index.ts
       ```
       This generates the systemd user unit at `~/.config/systemd/user/nanoclaw.service` with `Restart=on-failure` and enables linger.

    9. **Start NanoClaw via systemd:**
       ```
       systemctl --user start nanoclaw
       systemctl --user status nanoclaw
       ```

    10. **Verify linger is enabled (survives SSH logout):**
        ```
        loginctl show-user ec2-user | grep Linger
        ```
        Expected output: `Linger=yes`

    11. **Verify swap is active:**
        ```
        swapon --show
        free -h
        ```
        Expected: 1 GB swap shown.

    12. **Verify crash recovery by killing the process and checking it restarts:**
        ```
        systemctl --user status nanoclaw  # note the PID
        kill -9 $(systemctl --user show nanoclaw -p MainPID --value)
        sleep 5
        systemctl --user status nanoclaw  # should show active with a new PID
        ```
  </action>
